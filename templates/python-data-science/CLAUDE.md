# Claude Configuration - Python Data Science

## Project Context
Data science project using Python with common scientific libraries.

## Memory Patterns
- Remember data pipeline stages
- Track model hyperparameters
- Maintain reproducible notebook states
- Document data transformations

## Workflow Preferences
1. Exploratory data analysis (EDA) first
2. Data cleaning and preprocessing
3. Feature engineering
4. Model training and evaluation
5. Visualization and reporting

## Code Style Guidelines
- Follow PEP 8 conventions
- Use descriptive variable names for data
- Document functions with docstrings
- Separate configuration from logic

## Required Libraries
- pandas, numpy for data manipulation
- matplotlib, seaborn for visualization
- scikit-learn for machine learning
- jupyter for interactive development

## Notebook Conventions
- Markdown cells for section headers
- Code cells with clear explanations
- Limit cell output to essential visuals
- Restart and run all before committing

## Common Patterns
- Train/test split early
- Cross-validation for model selection
- Feature importance analysis
- Pipeline creation for reproducibility

## Hooks to Configure
- Auto-save notebook checkpoints
- Clear output before commit
- Export notebooks to scripts
- Data version tracking

## Avoid
- Hardcoded file paths
- Large data in version control
- Uncommented magic numbers
- Overfitting without validation
