# Claude Configuration - Documentation Generator

## Project Context
This project involves generating technical documentation, README files, and project documentation.

## Memory Patterns
- Remember documentation structure patterns
- Track API endpoint documentation
- Note code example patterns
- Remember branding and tone guidelines

## Workflow Instructions
1. **When documenting code**:
   - Document public functions and classes first
   - Include usage examples
   - Note parameters and return values
   - Document exceptions that may be raised

2. **When writing tutorials**:
   - Start with prerequisites
   - Break into logical steps
   - Include code snippets with explanations
   - Add troubleshooting sections

3. **When creating API docs**:
   - Document endpoints with method, path, parameters
   - Include request/response examples
   - Note authentication requirements
   - Document rate limits and quotas

4. **When writing READMEs**:
   - Start with project title and brief description
   - Include badges for build status, version, etc.
   - Add installation instructions
   - Include usage examples
   - Add contributing guidelines

## Documentation Style Guidelines
- Use active voice
- Keep sentences short and clear
- Use consistent terminology
- Include tables for complex information
- Use bullet points for lists
- Include diagrams for complex concepts

## Formatting Standards
- Use Markdown for most documentation
- Use reStructuredText for Python packages if needed
- Include table of contents for long documents
- Use consistent heading hierarchy

## Common Tasks
- Generate API documentation from code comments
- Create installation guides
- Write migration guides
- Document configuration options
- Create troubleshooting guides
- Write release notes
