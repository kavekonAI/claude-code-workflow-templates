# Contributing Guidelines

We welcome contributions to expand our collection of CLAUDE.md templates!

## How to Contribute

1. Fork the repository
2. Create a new template directory in `templates/your-template-name/`
3. Add your `CLAUDE.md` file with configuration
4. Include a brief description in a `README.md` within your template directory
5. Submit a pull request

## Template Requirements

Your CLAUDE.md template should include:

### Required Sections
1. **Project Context** - What type of project this is for
2. **Core Principles** - 3-5 guiding principles
3. **Code Style Guidelines** - Language/framework specific conventions
4. **Memory Patterns** - What Claude should remember across sessions
5. **Workflow Hooks** - Triggers and actions for common workflows

### Optional Sections
- Common Tasks
- Preferred Stack/Tools
- Things to Avoid
- Testing Guidelines
- Deployment Notes

## Quality Standards

- Templates should be practical and immediately useful
- Focus on specific domains or tech stacks
- Include real-world patterns, not just theoretical advice
- Keep templates concise (1-2 pages max)
- Use clear, actionable language

## Review Process

1. PR submitted
2. Maintainers review for:
   - Completeness of required sections
   - Practical utility
   - Clarity and organization
   - Uniqueness (not duplicating existing templates)
3. Feedback provided within 3 business days
4. Once approved, merged to main
